Does it matter to use relative size(%,em,rem) one styling the size of an element so that the element is responsive ?

Yes it does matter if we use “%”, “em” or “rem”. This is because these are relative units of measurement while pixel is an absolute unit of measurement. When we are trying to make a responsive web page, using pixel is a bad idea because it won’t change size for different devices with different view port sizes. The relative units vary for different sizes and everything is scaled up and down proportionally giving the page a more presentable look. There are some people that suggest the responsiveness can be done using media queries but using the relative units will prevent writing complex media queries and simplifies the work. You can read more about this on webFx.com.

In addition to this, using relative units makes the website adaptive to different setting changes by the user. So basically it is the choice between making a pixel exact control design or to design around changes and user control around their browsing experiences. The user needs to be able to zoom the viewport without loss of content or functionality. When users want to use browser font scaling feature by only changing font size and without zooming the whole page the difference is observed. If relative units are used for the font size specially rem, they are scaled up or scaled down to a multiple of the root level font. And also if the height of the container is also specified using relative units the line spacing is automatically adjusted giving a pleasant result. For detailed explanation and examples look at cssTricks.com.

What is the default browser size?

The default root element’s size for almost all browsers is 16px if you don’t apply any CSS rules. This value is used throughout the html when we use rem and %. This means 1rem is equal with 16px and also 100% is the same. When we set the size of a text or other element using pixels like for example when we set font-size as 200p, we are preventing the text from inheriting from the root and setting new values to that specific text. We can also change the value of the default root element’s size in our CSS by setting a different value to the html. To read further about this topic you can visit hongkiat.com.
